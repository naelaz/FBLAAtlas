# User Journey and UX Rationale

## Journey

1. **Login**
- User signs in with demo credentials.
- Validation prevents malformed email or weak password.

2. **Dashboard**
- User sees connectivity state and core navigation cards.
- High-priority paths are visible without scrolling.

3. **Profile**
- User edits role, interests, and accessibility preferences.
- Preferences persist and affect app-wide display.

4. **Calendar**
- User reviews events and sets reminders with lead-time options.
- Detail page clarifies date/time/location and reminder status.

5. **Resources + News**
- User consumes critical competition resources and chapter updates.
- Offline messaging explains fallback behavior.

6. **Social**
- User opens chapter channels directly in native apps when available.
- Fallback to web preserves continuity.

## UX Rationale

- Fast access to key tasks via dashboard cards.
- Drawer navigation gives predictable location awareness.
- Card layout and whitespace improve scanning during live demos.
- Accessibility settings reduce friction for different reading needs.
- Error and status messages are short, direct, and contextual.
